# 공공데이터포털 개발계정 인증키 설정 매뉴얼

> **대상 사이트:** [공공데이터포털](https://www.data.go.kr)  
> **작성일:** 2026-04-04  
> **목적:** Swagger UI에서 API 테스트 시 인증키를 설정하는 방법 안내

---

## 개요

공공데이터포털에서 Open API를 활용 신청하면 **개발계정 상세보기** 페이지에서 인증키를 확인할 수 있습니다.  
Swagger UI를 통해 API를 직접 테스트하려면 **인증키 설정**이 필요합니다.  
인증키는 두 가지 방식으로 적용됩니다:

- **ApiKeyAuth** – HTTP 헤더(`Authorization`)를 통한 인증
- **ApiKeyAuth2** – URL 쿼리 파라미터(`serviceKey`)를 통한 인증

---

## 사전 준비

1. 공공데이터포털 회원가입 및 로그인
2. 원하는 Open API에 활용 신청 완료 (처리상태: **승인**)
3. **마이페이지 > 데이터 활용 > Open API** 에서 해당 API 확인

---

## 인증키 확인 방법

1. 공공데이터포털 로그인 후 **[마이페이지]** 클릭
2. 좌측 메뉴에서 **데이터 활용 > Open API** 선택
3. 목록에서 원하는 API를 클릭하여 **개발계정 상세보기** 페이지 이동
4. **서비스정보** 섹션에서 **일반 인증키** 확인

| 항목 | 설명 |
|------|------|
| 데이터포맷 | JSON + XML |
| Base URL | `api.odcloud.kr/api` |
| Swagger URL | API 명세 확인 주소 |
| 일반 인증키 | API 호출 시 사용하는 인증키 (64자리 hex) |

> ⚠️ **참고:** API 환경 또는 호출 조건에 따라 Encoding/Decoding된 인증키를 적용해야 할 수 있습니다.

---

## Swagger UI 인증키 설정 방법

### Step 1. 개발계정 상세보기 페이지 접속

마이페이지에서 신청한 Open API를 클릭하여 **개발계정 상세보기** 페이지로 이동합니다.

---

### Step 2. Swagger UI 영역 확인

페이지 하단의 **활용신청 상세기능정보** 섹션으로 스크롤하면 Swagger UI가 표시됩니다.

```
[ 행정안전부_정부기능별분류체계 ]
Base URL: api.odcloud.kr/api
```

---

### Step 3. [인증키 설정] 버튼 클릭

Swagger UI 상단 우측에 있는 **[인증키 설정]** 버튼(🔓 자물쇠 아이콘)을 클릭합니다.

> 버튼 위치: Swagger UI 헤더 영역 우측 (Schemes 드롭다운 옆)

---

### Step 4. Available Authorizations 모달 확인

**Available authorizations** 팝업 창이 열리며, 두 가지 인증 항목이 표시됩니다:

#### ① ApiKeyAuth (apiKey)

| 항목 | 내용 |
|------|------|
| Name | `Authorization` |
| In | `header` |
| Value | *(인증키 입력란)* |

#### ② ApiKeyAuth2 (apiKey)

| 항목 | 내용 |
|------|------|
| Name | `serviceKey` |
| In | `query` |
| Value | *(인증키 입력란)* |

---

### Step 5. 인증키 입력

각 항목의 **Value** 입력란에 **개발계정 상세보기 페이지의 일반 인증키**를 붙여넣기 합니다.

```
예시 인증키 형식: 54cc7e12...f8b9227  (64자리 영문소문자+숫자)
```

> 💡 일반적으로 **두 항목 모두** 동일한 인증키를 입력합니다.

---

### Step 6. [설정] 버튼 클릭

각 항목 하단의 **[설정]** 버튼을 클릭합니다.  
(ApiKeyAuth의 설정 버튼 → ApiKeyAuth2의 설정 버튼 순서로 클릭)

설정이 완료되면 각 항목에 **"Authorized"** 문구가 표시됩니다.

```
ApiKeyAuth (apiKey)
✅ Authorized
Name: Authorization
In: header
Value: ******

ApiKeyAuth2 (apiKey)
✅ Authorized
Name: serviceKey
In: query
Value: ******
```

---

### Step 7. 모달 닫기

모달 우측 상단의 **✕** 버튼을 클릭하여 창을 닫습니다.

인증키 설정 버튼이 **🔒 잠금(locked)** 아이콘으로 변경된 것을 확인할 수 있습니다.

---

## 인증키 설정 후 API 테스트

1. Swagger UI의 **API 목록**에서 테스트할 엔드포인트를 클릭하여 펼칩니다.
2. **[Try it out]** 버튼 클릭
3. 필요한 파라미터 입력 (페이지 번호, 개수 등)
4. **[Execute]** 버튼 클릭
5. **Response** 섹션에서 API 응답 결과 확인

---

## 인증키 방식 선택 안내

| 방식 | 파라미터명 | 적용 위치 | 비고 |
|------|-----------|----------|------|
| ApiKeyAuth | `Authorization` | HTTP Header | 일부 API에서 사용 |
| ApiKeyAuth2 | `serviceKey` | URL Query | 공공데이터포털 표준 방식 |

> ⚠️ API 호출 시 오류가 발생하면 **Encoding된 인증키(URL 인코딩 적용)**를 사용해보세요.  
> 인코딩된 키는 `+`, `%2B` 등의 문자가 포함될 수 있습니다.

---

## 인증키 초기화 방법

인증키를 삭제하거나 변경하려면:

1. **[인증키 설정]** 버튼(🔒) 재클릭
2. 해당 항목의 **[설정 해제]** 버튼 클릭
3. 필요 시 새로운 키 입력 후 **[설정]** 버튼 클릭

---

## 자주 묻는 질문 (FAQ)

**Q. 인증키 설정 후에도 API 호출이 실패합니다.**  
A. Encoding/Decoding 인증키 구분을 확인하세요. 일반 인증키가 안 되면 URL 인코딩된 키를 시도해보세요.

**Q. 개발계정과 운영계정의 차이는 무엇인가요?**  
A. 개발계정은 테스트 및 개발 목적이며, 일 호출 횟수 제한이 있을 수 있습니다. 서비스 배포 시에는 운영계정 신청이 필요합니다.

**Q. 인증키는 어디서 다시 확인할 수 있나요?**  
A. 마이페이지 > 데이터 활용 > Open API > 해당 API 클릭 > 개발계정 상세보기 > 서비스정보 > 일반 인증키
