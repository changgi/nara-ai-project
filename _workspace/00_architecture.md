# NARA-AI 아키텍처 설계서

## 프로젝트 비전

> "국민을 위한, 국민에 의한, 국민에게 혜택이 돌아가는" AI 기록물 관리 시스템

290만 전자기록물 + 100만 페이지 비전자기록물에 AI를 적용하여:
- 국민이 자연어로 국가기록에 접근
- 기록물 전문가의 업무를 10배 효율화
- 비공개 기록물의 체계적 공개 전환 지원

## 11단계 처리 파이프라인

```
1.수집 → 2.레이아웃 → 3.OCR → 4.후처리 → 5.분류 → 6.메타데이터
   ↓                                                    ↓
   7.비밀해제[HITL] → 8.임베딩 → 9.그래프 → 10.품질 → 11.보안
```

## 처리 목표

| 항목 | 목표 | GPU 요구 |
|------|------|---------|
| 기록물 분류 | F1 ≥ 0.92 | 2 GPU (추론) |
| 메타데이터 생성 | ROUGE-1 ≥ 0.85 | 2 GPU (추론) |
| OCR (활자) | CER ≤ 3% | 1 GPU |
| OCR (필사) | CER ≤ 10% | 1 GPU |
| OCR (한자혼용) | CER ≤ 7% | 1 GPU |
| 시맨틱 검색 | Recall@10 ≥ 0.90, P99 ≤ 2s | 1 GPU (임베딩) |
| 비밀해제 심사 | Precision ≥ 0.95 (PII) | 2 GPU (추론) |

## 기존 자산 통합

10개 프로젝트 (160K+ LOC, 279 API, 165 에이전트)를 MCP 프로토콜로 통합:
- 신규 코드 작성보다 기존 자산의 MCP 래핑을 우선
- 예상 절감 효과: 12-18개월 개발 기간 단축

## 법적 준수

1. **AI 기본법** (2026.1 시행): 위험평가, 투명성, 인간 개입, 편향 방지
2. **ISMS-P** (2027.7 의무): 101개 항목, JWT+RBAC, 암호화, PII 가명처리
3. **N2SF**: "민감" 등급, 물리적 에어갭
4. **공공기록물법**: 제6,33-35,38조 준수, HITL 게이트 3개
