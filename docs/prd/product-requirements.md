# NARA-AI 제품요구사항서 (PRD)

## 1. 제품 비전

> 국민을 위한, 국민에 의한, 국민에게 혜택이 돌아가는 AI 기록물 관리 시스템

## 2. 대상 사용자

| 사용자 | 역할 | 핵심 니즈 |
|--------|------|----------|
| 국민 | 기록물 열람/검색 | 자연어로 국가기록에 쉽게 접근 |
| 기록물관리사 | 분류/정리/심사 | 반복 업무 자동화, AI 지원 의사결정 |
| 연구자 | 심층 조사 | 다국어 검색, 관계 탐색, 시계열 분석 |
| 관리자 | 시스템 운영 | 모니터링, 성능 관리, 보안 감사 |

## 3. 핵심 기능 요구사항

### FR-01: 자연어 검색
- 국민이 한국어 자연어로 기록물을 검색한다
- 시맨틱 검색 + 키워드 검색 + 그래프 탐색 결합
- P99 레이턴시 ≤ 2초
- 보안등급 기반 접근 제어

### FR-02: 기록물 자동 분류
- BRM 16개 대분류 자동 매핑
- F1 Score ≥ 0.92
- 신뢰도 0.7 미만 시 전문가 검수 의뢰 (HITL)

### FR-03: 메타데이터 자동 생성
- 제목, 요약(200자), 키워드(5-10개), NER 자동 추출
- ROUGE-1 ≥ 0.85
- ISAD(G) 기술 표준 준수

### FR-04: OCR 디지털화
- 활자체: CER ≤ 3%
- 필사체: CER ≤ 10%
- 한자혼용: CER ≤ 7%
- 처리량: 1,500-3,800 페이지/시간 (8 GPU)

### FR-05: 비밀해제 심사 지원
- PII 자동 탐지 (6종, Precision ≥ 0.95)
- AI 공개 적합성 평가 + 법적 근거 제시
- 최종 결정은 반드시 인간이 수행 (HITL)

### FR-06: 감사추적
- 모든 AI 의사결정의 근거와 이력을 기록
- 10년 보존 (공공기록물법)
- AI 기본법 투명성 요구사항 충족

## 4. 비기능 요구사항

### NFR-01: 보안
- N2SF "민감" 등급 물리적 에어갭
- ISMS-P 101개 항목 준수
- 4중 보안 스캐닝 (CodeQL, Semgrep, Trivy, TruffleHog)

### NFR-02: 법률 준수
- AI 기본법 (2026.1): 위험평가, 투명성, 인간개입, 편향방지
- 공공기록물법: 제6/33/34/35/38조
- 개인정보보호법: PII 가명처리 (제28조의2)

### NFR-03: 확장성
- 8 GPU MVP → 32 GPU 확장 (설정 변경만으로)
- 290만 전자기록 → 전체 국가기록 처리 가능

### NFR-04: 가용성
- 서비스 가용성 99.9%
- 자동 복구 (K8s liveness/readiness probe)
- GPU 장애 시 워크로드 재배정

## 5. 성공 지표

| 지표 | 현재 | 목표 |
|------|------|------|
| 기록물 검색 시간 | 수십 초 (키워드) | ≤ 2초 (자연어) |
| 분류 처리량 | 수작업 | 자동 (F1 ≥ 0.92) |
| OCR 디지털화 | 30K건/년 | 100만 페이지/28일 |
| 비밀해제 심사 | 수작업 개별 검토 | AI 추천 + HITL |
| 국제 벤치마크 | US NARA 14개 대비 부족 | 동등 이상 |
